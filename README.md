# Flutter AI Tutor WhatsApp Bot 🚀

هذا المشروع عبارة عن بوت واتساب يعمل بالذكاء الاصطناعي لتعليم لغة **Flutter**، تم تصميمه ليعمل مع **Evolution API**، ويعتمد على تقنية **RAG (Retrieval-Augmented Generation)** باستخدام قاعدة بيانات **Supabase** مع ميزة المتجهات (pgvector). ويعتمد البوت على **Groq AI** لسرعة صياغة وتوليد المحتوى بذكاء.

## المميزات الأساسية ✨
يحقق البوت المهام العشرة التالية بناءً على متطلبات مشروع التخرج/البحث:
1. **تسجيل بيانات الطالب**: حفظ بيانات ومستوى كل طالب تلقائياً.
2. **اختبار تحديد المستوى**: اختبار ذكي للمشتركين الجدد.
3. **دروس مخصصة**: تقديم دروس متسلسلة حسب مستوى كل مستخدم (تُجلب من قاعدة البيانات).
4. **اختبارات مستمرة**: تقييم فهم الطالب بعد كل درس.
5. **تصحيح الأخطاء البرمجية**: التدخل لحل المشاكل البرمجية التي تواجه الطالب وتقديم شرح مفصل.
6. **مساعدة في كتابة الكود**: شرح المفاهيم والأكواد بالتفصيل.
7. **تحويل المنطق إلى كود**: الاستماع لطريقة تفكير الطالب وتحويلها لكود Flutter.
8. **مسار تعليمي ذكي (Adaptive Learning)**: تغيير مستوى ومسار الطالب بناءً على إجاباته في الاختبارات.
9. **نظام التلعيب (Gamification)**: استخدام النقاط (Points) والتشجيع المستمر لتحفيز المستخدم.
10. **واجهة أدمن / مراقبة**: إمكانية تتبع الأداء الإجمالي عبر مسار `/admin/stats`.

## الإعدادات المطلوبة (Environment Variables) ⚙️
قبل تشغيل البوت أو رفعه على **Render**، يجب إضافة المتغيرات التالية:

```env
# إعدادات Supabase (قاعدة البيانات)
SUPABASE_URL=your_supabase_url
SUPABASE_KEY=your_supabase_service_role_key

# إعدادات Evolution API (واتساب)
EVOLUTION_API_URL=your_evolution_api_url
EVOLUTION_API_KEY=your_evolution_api_key
EVOLUTION_INSTANCE_NAME=your_instance_name

# إعدادات الذكاء الاصطناعي والـ Embeddings
GROQ_API_KEY=your_groq_api_key
HUGGINGFACE_TOKEN=your_huggingface_token
```

## إعداد قاعدة بيانات Supabase (SQL) 🗄️
لكي يعمل نظام **RAG** بشكل صحيح، يجب تشغيل سكربت SQL التالي في محرر الـ SQL داخل لوحة تحكم Supabase الخاص بك:

```sql
create extension if not exists vector;

-- جدول الطلاب
create table students (
  id bigint generated by default as identity primary key,
  phone_number text unique,
  name text,
  level int default 0,
  points int default 0,
  current_state text
);

-- جدول الدروس
create table lessons (
  id bigint generated by default as identity primary key,
  level int not null,
  content text not null,
  embedding vector(384)
);

-- دالة البحث التلقائي للـ RAG
create or replace function match_lessons (
  query_embedding vector(384),
  match_threshold float,
  match_count int,
  student_level int
)
returns table (
  id bigint,
  content text,
  similarity float
)
language sql stable
as $$
  select
    id,
    content,
    1 - (lessons.embedding <=> query_embedding) as similarity
  from lessons
  where 1 - (lessons.embedding <=> query_embedding) > match_threshold
    and level = student_level
  order by similarity desc
  limit match_count;
$$;
```

## التشغيل محلياً 💻
1. قم بإنشاء بيئة وهمية (Virtual Environment) وتفعيلها:
   ```bash
   python -m venv venv
   source venv/bin/activate  # Mac/Linux
   venv\Scripts\activate     # Windows
   ```

2. تثبيت المكتبات:
   ```bash
   pip install -r requirements.txt
   ```

3. تشغيل الخادم محلياً عبر Uvicorn:
   ```bash
   uvicorn main:app --reload --port 8000
   ```

## الرفع على منصة Render ☁️
تم إعداد المشروع ليكون متوافقاً تماماً مع Render بفضل ملف `Procfile` الجاهز.
1. قم برفع مجلد المشروع إلى **GitHub**.
2. في Render، اختر **Web Service** واربط مستودعك.
3. تأكد من تحديد **Start Command** كالتالي (إذا لم يقرأ الـ Procfile تلقائياً):
   ```bash
   uvicorn main:app --host 0.0.0.0 --port $PORT
   ```
4. أضف كافة **Environment Variables** المذكورة أعلاه في إعدادات التطبيق على Render.
